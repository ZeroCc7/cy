# 前端路由重设计

## 目标

将入口从"直接进入聊天"改为"项目列表优先"，使用 Hash 路由实现两个视图的切换，并优化聊天框的长文本体验。

---

## 路由结构

| Hash | 视图 | 说明 |
|---|---|---|
| `#/` | 首页 | 项目列表，默认入口 |
| `#/project/new` | 创作页 | 新建项目，空白流程 |
| `#/project/:id` | 创作页 | 打开已有项目 |

**实现方式：**
- `window.addEventListener('hashchange', router)` 监听路由变化
- `goto(hash)` 统一跳转，内部执行 `location.hash = hash`
- 页面初始化时调用 `router()` 根据当前 hash 渲染对应视图
- 两个视图共用 `<body>`，切换时替换主内容区 DOM

## 首页（项目列表）

**布局：**
```
┌─────────────────────────────────────────────┐
│  ✦ 短视频剧本生成器              [+ 新建剧本] │
├─────────────────────────────────────────────┤
│  全部项目 (n 个)                             │
│                                             │
│  ┌──────────────┐  ┌──────────────┐  ...    │
│  │ 《标题》     │  │ 《标题》     │         │
│  │ ④ 剧本      │  │ ③ 大纲       │         │
│  │ 1/15 集     │  │  —           │         │
│  │ 05-23  [×]  │  │ 05-22  [×]   │         │
│  └──────────────┘  └──────────────┘         │
│                                             │
│  ── 空状态 ──                               │
│  还没有剧本，点击右上角「新建」开始创作       │
└─────────────────────────────────────────────┘
```

**项目卡片字段：** 标题、阶段徽章、集数进度（已完成/总集数）、最后更新时间、删除按钮

**删除：** 点击 `[×]` 弹确认后调 `DELETE /api/project/:id`，后端从 Supabase 删除记录，前端移除卡片

**空状态：** 无项目时显示引导文案，不显示卡片区

**首页 Header：** 只有 logo + 「新建剧本」按钮，无步骤条、无「项目历史」

## 创作页

**创作页 Header：**
- 最左侧：「← 首页」按钮，点击执行 `goto('#/')`
- 中间：步骤条（① 聊需求 → ② 世界观 → ③ 大纲 → ④ 剧本）
- 右侧：「项目历史」按钮（历史面板保留"查看/修改内容"功能）

**AI 欢迎语时机：**
- 移除现有 `window.onload` 里的自动问候
- 改为进入 `#/project/new` 时触发，确保每次新建都有欢迎语，刷新首页不会误触发

**自动生成项目名：**
1. 用户发出第一条消息时，立即以消息前 20 字为临时标题，调 `POST /api/project` 在 Supabase 创建记录并拿到 `projectId`
2. 世界观生成完毕后（已有大纲标题），用 `《书名》` 更新 `title` 字段

**聊天框长文本支持：**
- 去掉 `max-height: 110px` 限制，改为 `max-height: 40vh`
- 移除 `keydown` 里的 Enter 发送逻辑，Enter 键只换行
- 发送唯一方式：点击发送按钮

| 方法 | 路径 | 说明 |

## 不变的部分

- 创作流程四阶段逻辑（chat → worldbuilding → outline → episodes）不变
- SSE 流式生成不变
- 历史面板的"查看/修改/重新生成"功能不变
- Supabase 数据结构不变
